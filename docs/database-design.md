# Проектирование базы данных

## Общая информация

База данных предназначена для хранения информации о пользователях, книгах, заказах, оплате, наличии товаров в магазинах и обращениях в службу поддержки.

---

## Users

Хранит данные зарегистрированных пользователей системы.

Поля:

* id — идентификатор пользователя;
* email — электронная почта;
* password_hash — зашифрованный пароль;
* first_name — имя;
* last_name — фамилия;
* role — роль пользователя;
* created_at — дата регистрации.

Роли пользователей:

* USER — обычный пользователь;
* SUPPORT — сотрудник поддержки;
* ADMIN — администратор.

---

## Books

Хранит информацию о книгах.

Поля:

* id — идентификатор книги;
* title — название;
* author — автор;
* isbn — международный номер книги;
* description — описание;
* price — стоимость;
* preview_url — ссылка на предпросмотр;
* publication_year — год публикации;
* created_at — дата добавления.

---

## Categories

Содержит список категорий книг.

Поля:

* id — идентификатор категории;
* name — название категории.

Примеры категорий:

* Фантастика;
* Детектив;
* Научная литература;
* История.

---

## BookCategories

Промежуточная таблица для связи книг и категорий.

Одна книга может относиться к нескольким категориям, а одна категория может содержать множество книг.

Поля:

* book_id;
* category_id.

---

## Stores

Содержит информацию о физических магазинах сети.

Поля:

* id — идентификатор магазина;
* name — название магазина;
* address — адрес.

---

## Inventory

Содержит данные о наличии книг в магазинах.

Поля:

* id — идентификатор записи;
* store_id — магазин;
* book_id — книга;
* quantity — количество экземпляров.

Позволяет узнать наличие книги в конкретном магазине.

---

## Orders

Хранит информацию о заказах пользователей.

Поля:

* id — идентификатор заказа;
* user_id — покупатель;
* status — статус заказа;
* total_price — общая стоимость;
* created_at — дата создания.

Возможные статусы:

* CREATED;
* PAID;
* SHIPPED;
* DELIVERED;
* CANCELLED.

---

## OrderItems

Содержит состав заказа.

Поля:

* id — идентификатор позиции;
* order_id — заказ;
* book_id — книга;
* quantity — количество;
* price — цена на момент покупки.

---

## PaymentMethods

Содержит доступные способы оплаты.

Поля:

* id — идентификатор;
* name — название способа оплаты.

Примеры:

* Банковская карта;
* СБП
* SberPay.

---

## Payments

Хранит информацию об оплате заказов.

Поля:

* id — идентификатор платежа;
* order_id — заказ;
* payment_method_id — способ оплаты;
* amount — сумма;
* status — статус;
* created_at — дата оплаты.

---

## SupportTickets

Хранит обращения пользователей в службу поддержки.

Поля:

* id — идентификатор обращения;
* user_id — автор обращения;
* subject — тема обращения;
* status — статус;
* created_at — дата создания.

Статусы:

* OPEN;
* IN_PROGRESS;
* CLOSED.

---

## SupportMessages

Содержит переписку внутри обращения.

Поля:

* id — идентификатор сообщения;
* ticket_id — обращение;
* sender_id — отправитель;
* message — текст сообщения;
* created_at — дата отправки.

---

## AuditLog

Содержит журнал действий пользователей.

Поля:

* id — идентификатор записи;
* user_id — пользователь;
* action — выполненное действие;
* created_at — дата и время.

Примеры действий:

* вход в систему;
* создание заказа;
* оплата заказа;
* создание обращения в поддержку.
